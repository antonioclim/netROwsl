Concept Application ‚Üí [User Data] Transport ‚Üí [TCP/UDP Header | Data] = Segment/Datagram Network ‚Üí [IP Header | Segment] = Packet Link ‚Üí [Antet Ethernet | Packet | CRC] = Frame

ü§î Think Before You Answer

What is the minimum total overhead for a TCP packet? (Ethernet + IP + TCP)

14 bytes (Ethernet) + 20 bytes (IP minimum) + 20 bytes (TCP minimum) = 54 bytes minimum

For a 1-byte payload, that's ~98% overhead! This is why Nagle's algorithm combines small messages and why you should batch small writes.

üî¢ Address Types ‚Äî Each Layer Has Its Own

Layer Tip AdresƒÉ Size Exemplu Resolution
L2 (Link) MAC 48 bits aa:bb:cc:dd:ee:ff ARP
L3 (Network) IP 32 / 128 bits 192.168.1.100 DNS, routing
L4 (Transport) Port 16 bits 80, 443, 22 Application

Key insight: A full socket address combines IP + Port (e.g., 192.168.1.100:8080). The combination uniquely identifies a process endpoint on the Internet.

üîç ARP ‚Äî The Bridge Between L2 and L3

Address Resolution Protocol (ARP)

A protocol that maps Network layer addresses (IPv4) to Data Link layer addresses (MAC) within a local network segment. It enables IP packets to be encapsulated in Ethernet frames for transmission.

How ARP Works

Request (broadcast):
"Who has IP 10.0.0.2? Tell 10.0.0.1"

Reply (unicast):
"10.0.0.2 is at aa:bb:cc:dd:ee:ff"

Verification:

ip neigh or arp -a
‚ö†Ô∏è Security Vulnerability

ARP has no authentication. Anyone on the LAN can send false ARP replies and "poison" neighbours' caches‚Äîthe basis of Man-in-the-Middle attacks on unsecured Wi-Fi networks (CVE-1999-0667).

üìú Historical Incident: Panix SYN Flood (1996)

The first major documented DDoS attack: attackers sent avalanches of fake SYN packets, exhausting the connection table. The solution? SYN Cookies‚Äîthe server encodes connection state in the TCP sequence number itself, eliminating the need to store state for incomplete handshakes.

What RST means

An RST (Reset) packet indicates something went wrong:

  • Port not listening
  • Connection rejected by firewall
  • Application crashed
  • Invalid segment received

üó∫Ô∏è Routing ‚Äî How Packets Find Their Way

Routing Table Components

  • UrmƒÉtorul-hop: Where to forward
  • Interface: Which NIC to use
  • Metric: Cost/preference
Bash $ ip route default via 192.168.1.1 dev eth0 10.0.14.0/24 dev mn-h1 scope link 172.16.0.0/16 via 10.0.14.1 dev eth0

Longest Prefix Match

The most specific route wins.

Exemplu: For destination 10.0.0.5:

  • 0.0.0.0/0 ‚Üí matches (/0)
  • 10.0.0.0/8 ‚Üí matches (/8)
  • 10.0.0.0/24 ‚Üí CHOSEN (/24)

üßÆ CIDR Quick Reference

Network mask: 255.255.255.0

Total addresses: 256

Usable hosts: 254

/24 = 256 addresses, 254 usable hosts (most common for small networks)

/16 = 65,536 addresses (traditional Class B network)

üåê BGP ‚Äî When Routing Goes Wrong

‚ö° Incident: YouTube-Pakistan (2008)

Pakistan Telecom attempted to block YouTube locally by announcing a more specific BGP prefix for YouTube's IP range. The announcement leaked through PCCW to the global Internet. For approximately 2 hours, YouTube traffic worldwide was black-holed into Pakistan's network. BGP, by design, has no built-in verification of announcement legitimacy.

Lesson Learned

Modern networks depend on trust and filtering. Initiatives like RPKI (Resource Public Key Infrastructure) add cryptographic validation to BGP announcements, but adoption remains incomplete.

üî§ DNS ‚Äî From Names to Addresses

üèõÔ∏è Context Istoric

Until 1983, name-to-IP translation was done via a single HOSTS.TXT file, manually copied to every computer on ARPANET. Paul Mockapetris proposed DNS‚Äîa distributed, hierarchical database with caching. The first version had only 7 top-level domains (.com, .edu, .gov, .mil, .net, .org, .arpa).

Bash # Query A record $ dig example.com +short 93.184.216.34 # Query with full details $ dig example.com # Use specific DNS server $ dig @8.8.8.8 example.com

‚ö†Ô∏è DNS as a Failure Point

Demo # Working DNS $ curl http://example.com/ ... # Broken DNS (bad nameserver) $ echo "nameserver 1.2.3.4" | sudo tee /etc/resolv.conf $ curl http://example.com/ curl: (6) Could not resolve host: example.com # Bypass DNS with direct IP $ curl http://93.184.216.34/ ...
Sfat de Diagnostic

If curl/wget fails with "could not resolve host" but works with an IP address, DNS is the problem‚Äînot the network or the server.

Load balancing and reverse proxies in practice

‚öñÔ∏è Pattern: Load Balancer / Reverse Proxy

In the starterkit: WEEK14/python/apps/lb_proxy.py ‚Äî a complete Python implementation of round-robin load balancing with passive health checks.

üíª Load Balancer: Key Algorithm

Python # From lb_proxy.py ‚Äî Round-robin distribution def get_next_backend(self) -> Optional[Backend]: """Returns next healthy backend (round-robin).""" with self._lock: healthy = [b for b in self.backends if b.healthy] if not healthy: healthy = self.backends # Fallback: try any if not healthy: return None backend = healthy[self._current_index % len(healthy)] self._current_index = (self._current_index + 1) % len(healthy) return backend
Passive Health Check

After 3 consecutive failures, a backend is marked unhealthy: backend.mark_failure(). Success resets the counter: backend.mark_success().

üì® Proxy Headers ‚Äî Preserving Client Identity

Why Headers Matter

When a proxy forwards requests, the backend sees the proxy's IP, not the client's. Standard headers preserve original client information:

  • X-Forwarded-For: Client IP chain
  • X-Real-IP: Original client IP
  • X-Forwarded-Host: Original Host header
  • X-Forwarded-Proto: http or https
Python # Adding headers in lb_proxy.py client_ip = self.client_address[0] headers["X-Forwarded-For"] = client_ip headers["X-Real-IP"] = client_ip headers["X-Forwarded-Host"] = \ self.headers.get("Host", "")
