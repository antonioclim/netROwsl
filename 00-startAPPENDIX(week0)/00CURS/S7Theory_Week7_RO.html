Algoritmul lui Dijkstra (SPF ‚Äî Cea Mai ScurtƒÉ Cale √ént√¢i): A greedy algorithm that computes the shortest paths from a single source node to all other nodes in a weighted graph with non-negative edge weights.

üìå Key Properties

  • Greedy: Always selects the unvisited node with minimum distance
  • Optimal: Produces shortest path tree (SPT)
  • Requirement: All edge weights must be ‚â• 0
  • Output: Distance and predecessor arrays

‚è±Ô∏è Complexitate

  • Array-based: O(V¬≤)
  • Binary heap: O((V + E) log V)
  • Fibonacci heap: O(V log V + E)

For dense graphs (E ‚âà V¬≤), array is competitive. For sparse graphs, heap-based wins.

Why Non-Negative Weights? Dijkstra assumes that once a node is "settled" (extracted from the priority queue), its distance is final. Negative edges could later provide a shorter path, violating this assumption.

Dijkstra ‚Äî Pseudocod

Algoritmul Bellman-Ford

Funda»õia RutƒÉrii Distance-Vector

A simpler algorithm that trades speed for flexibility ‚Äî and introduces new challenges.

Bellman-Ford ‚Äî Prezentare GeneralƒÉ

Bellman-Ford Equation:
D(x, y) = min‚àÄv { c(x, v) + D(v, y) } The distance from x to y equals the minimum of: (cost from x to neighbour v) + (v's advertised distance to y), taken over all neighbours v.

‚úÖ Avantaje

  • Simple to implement and understand
  • Low memory: O(V) per router
  • Works with negative edge weights
  • Detects negative cycles
  • Distributes naturally across routers

‚ùå Dezavantaje

  • Convergen»õƒÉ lentƒÉ: O(V √ó E) iterations
  • Problema count-to-infinity
  • Temporary routing loops possible
  • Limited scalability (RIP: max 15 hopuri)
  • Periodic updates consume bandwidth

Bellman-Ford ‚Äî Pseudocod

function BellmanFord(Graph G, source s): // INITIALISATION for each vertex v in G.vertices: dist[v] ‚Üê ‚àû prev[v] ‚Üê undefined dist[s] ‚Üê 0 // RELAXATION (V-1 iterations) for i from 1 to |G.vertices| - 1: for each edge (u, v, w) in G.edges: if dist[u] + w < dist[v]: dist[v] ‚Üê dist[u] + w prev[v] ‚Üê u // NEGATIVE CYCLE DETECTION for each edge (u, v, w) in G.edges: if dist[u] + w < dist[v]: return "Negative cycle detected" return dist, prev
Why V-1 iterations? In the worst case, the shortest path uses V-1 edges. Each iteration propagates distances by one hop.
In RIP: This algorithm runs distributedly ‚Äî each router exchanges distance vectors with neighbours every 30 secunde.

Problema Count-to-Infinity

BEFORE: A ‚îÄ‚îÄ‚îÄ(1)‚îÄ‚îÄ‚îÄ B ‚îÄ‚îÄ‚îÄ(1)‚îÄ‚îÄ‚îÄ C All routers know: A reachable at cost 1 (from B) or 2 (from C) AFTER: A ‚úó B ‚îÄ‚îÄ‚îÄ(1)‚îÄ‚îÄ‚îÄ C Link A-B fails! Iteration 1: B: "I lost A directly... but C says it can reach A (cost 2)" B updates: dist[A] = 1 + 2 = 3 (via C) Iteration 2: C: "B now says A is cost 3... so I can reach A at cost 4" C updates: dist[A] = 1 + 3 = 4 (via B) Iteration 3: B: "C says A is cost 4... I can do 5" ... continues until metric reaches infinity (16 in RIP)
Root Cause: Stale information bounces back and forth. B uses C's old information about A, and C uses B's derived (wrong) information.

Atenuarea Count-to-Infinity

Never advertise a route back to the neighbour from which you learned it.

B learned A from A B will NOT advertise A to A

‚ò†Ô∏è Poison Reverse

Advertise the route with metric = infinity to the source neighbour.

B learned A from A B advertises to A: "A cost = ‚àû"

‚è±Ô∏è Hold-Down Timer

After a route is marked unreachable, ignore updates for a period (180s in RIP).

Allows time for the failure to propagate across the network.

‚ö° ActualizƒÉri Declan»ôate

Send updates immediately when a route changes, don't wait for the periodic timer.

Reduces convergence time significantly.

Primul Mare Atac DDoS

September 1996 ‚Äî Panix Under Siege

"Panix, one of New York's oldest Internet service providers, found itself overwhelmed by a flood of SYN packets. The attack, later identified as a SYN flood, exhausted server resources by initiating thousands of half-open TCP connections. üìâ Attack Mechanism

üõ°Ô∏è Defence Born

Lesson: Understanding TCP's three-way handshake is essential for both implementing servers and defending them.

Filtrare Pachete

The foundation of network firewalls

Packet Filter: A network device (or software) that examines packet headers and makes forwarding decisions based on a set of rules. Actions include ACCEPT, DROP, REJECT, and LOG.

üìã Filtering Criteria

  • Source/Destination IP address
  • Source/Destination port
  • Protocol (TCP, UDP, ICMP)
  • TCP flags (SYN, ACK, FIN, RST)
  • Interface (ingress/egress)
  • Connection state

‚õìÔ∏è Lan»õuri iptables

  • INPUT: Packets destined for local host
  • OUTPUT: Packets generated by local host
  • FORWARD: Packets being routed through
  • PREROUTING: Before routing decision
  • POSTROUTING: After routing decision
First-Match Wins: Rules are evaluated in order. The first matching rule determines the action. Order matters critically!

Anatomia unei Reguli iptables

iptables -A INPUT -p tcp --dport 22 -s 192.168.1.0/24 -j ACCEPT
Component Meaning
-A INPUT Append rule to the INPUT chain
-p tcp Match TCP protocol
--dport 22 Match destination port 22 (SSH)
-s 192.168.1.0/24 Match source network
-j ACCEPT Jump to ACCEPT target (allow packet)
Common Pattern: Allow specific services, then DROP or REJECT all else as the final rule.

Inspec»õie Stateful a Pachetelor

Stateful Filtering: Tracks the state of network connections and makes decisions based on connection context, not just individual packets.
# Allow established and related connections iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT # Allow new SSH from specific subnet iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/8 -m conntrack --ctstate NEW -j ACCEPT # Drop invalid packets iptables -A INPUT -m conntrack --ctstate INVALID -j DROP
State Description
NEW First packet of a connection (e.g., SYN)
ESTABLISHED Part of an existing connection
RELATED Related to existing connection (e.g., FTP data)
INVALID Does not match any known connection

Sisteme de Detectare a Intruziunilor (IDS)

IDS: A system that monitors network traffic or host activity for malicious behaviour or policy violations. Can be network-based (NIDS) or host-based (HIDS).

üìù Signature-Based

Matches traffic against known attack patterns.

  • ‚úì Low false positives for known attacks
  • ‚úì Fast and efficient
  • ‚úó Cannot detect zero-day attacks
  • ‚úó Requires constant signature updates

Exemplus: Snort, Suricata

üìä Anomaly-Based

Learns "normal" behaviour and flags deviations.

  • ‚úì Can detect unknown attacks
  • ‚úì Adapts to environment
  • ‚úó Higher false positives
  • ‚úó Training period required

Exemplus: ML-based systems, UEBA

IDS vs IPS

IDS: Passive monitoring. Alerts administrators but does not block traffic. Good for visibility without risk of false positive disruptions.
IPS: Inline blocking. Can stop attacks in real-time but may cause disruptions if rules are too aggressive.

Link-State (OSPF)
  • Global topology view via LSA flooding

  • Fast convergence, loop-free

  • Bellman-Ford equation
  • Simple but slow convergence
  • Problema count-to-infinity
