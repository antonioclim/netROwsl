FTP ‚Äì Protocol de Transfer Fi»ôiere

Dual-connection architecture for efficient file transfers between heterogeneous systems

Ce este FTP?

Defini»õie

FTP (Protocol de Transfer Fi»ôiere) is an application-layer protocol designed for transferring files between a client and server over a TCP network, using separate connections for control commands and data transfer.

Caracteristici Cheie

  • Operates over TCP (reliable delivery)
  • Uses two connections: control and data
  • Supports authentication and anonymous access
  • ASCII commands, binary/text file transfers

Context Istoric

FTP was first specified in RFC 114 (1971), making it one of the oldest Internet protocols still in use. The modern specification is RFC 959 (1985).

Security Aten»õie

Classic FTP transmits credentials in plaintext. Modern deployments should use FTPS (FTP over TLS) or SFTP (SSH Protocol de Transfer Fi»ôiere).

FTP Dual-Connection Architecture

Unlike most protocols, FTP separates command signalling from data transfer:

Why Two Connections?

Separation allows out-of-band signalling‚Äîyou can abort a transfer without losing the session. It also enables concurrent transfers and directory listings during active downloads.

Active vs Mod Pasiv

The critical distinction that determines firewall traversal:

üî¥ Mod Activ (PORT)

  1. Client opens a local port
  2. Client sends PORT ip,ip,ip,ip,p1,p2
  3. Server initiates connection from port 20

üü¢ Mod Pasiv (PASV)

  1. Client sends PASV
  2. Server responds with IP and port
  3. Client initiates data connection

Advantage

SECTION 02

DNS ‚Äì Sistem de Nume de Domeniu

The hierarchical, distributed naming system that makes the Internet human-readable

The Problem DNS Solves

Imagine the Alternative

Without DNS, browsing would require memorising: 142.250.180.206 for Google, 157.240.1.35 for Facebook...

Defini»õie

DNS (Sistem de Nume de Domeniu) is a hierarchical, distributed database that translates human-readable domain names into IP addresses.

ProprietƒÉ»õi Cheie

  • Distributed: No single point of failure
  • Hierarchical: Organised in a tree structure
  • Cached: Multi-level caching for performance

Protocol Details

  • Primary transport: UDP port 53
  • TCP for zone transfers and large responses
  • Specified in RFC 1034/1035 (1987)

Why UDP?

DNS queries are typically small (< 512 bytes) and benefit from UDP's low latency. If truncated (TC flag), the client retries over TCP.

Ierarhia DNS

Servere Root

13 logical servers (a.root-servers.net through m.root-servers.net), replicated globally via anycast.

Servere TLD

Managed by registries (Verisign for .com, ROTLD for .ro). Know authoritative servers for each domain.

Rezolu»õie DNS: Step by Step

Iterative vs Recursive Resolution

The recursive resolver does the heavy lifting‚Äîquerying root, TLD, and authoritative servers. The client's stub resolver simply asks the recursive resolver and waits.

DNS Security: Vulnerabilities and Solutions

üéØ DNS Spoofing

Attacker injects false responses before legitimate server replies. Exploits UDP's lack of authentication.

üîç DNS Hijacking

Malicious resolver returns wrong IPs. Often done by ISPs for "helpful" error pages or censorship.

üîê DNSSEC

Cryptographic signatures on DNS records. Chain of trust from root to authoritative. Validates authenticity.

üõ°Ô∏è DoH / DoT

DNS over HTTPS (port 443) and DNS over TLS (port 853). Encrypts queries to prevent eavesdropping.

Why isn't DNSSEC universal?

Complexity of key management, increased packet sizes, performance overhead, and lack of incentive. ~90% of TLDs support DNSSEC, but only ~30% of .com domains are signed.

SECTION 03

SSH ‚Äì Secure Shell

Encrypted remote access, authentication, and secure tunnelling

What is SSH?

Defini»õie

SSH (Secure Shell) is a cryptographic network protocol for secure data communication, remote command execution, and other network services between networked computers.

Context Istoric

SSH was created in 1995 by Tatu Yl√∂nen after a password-sniffing attack on his university network. It replaced Telnet, rsh, rlogin.

SSH Provides

  • Autentificare: Password, public key, certificates
  • Confidentiality: AES, ChaCha20 encryption
  • Integrity: HMAC verification
  • Port forwarding: Secure tunnels

Protocol Details

  • Default port: TCP 22
  • Specification: RFC 4251‚Äì4254
  • Current version: SSH-2

SSH Protocol Architecture

Nivelul Transport

Handles encryption negotiation, key exchange (Diffie-Hellman), and server authentication via host keys.

Nivelul Conexiune

Multiplexes multiple logical channels over the encrypted transport‚Äîsimultaneous shell, file transfer, and port forwarding.

SSH Key Exchange

First Connection Aten»õie

The "authenticity of host can't be established" message appears on first connection. The client saves the host key to ~/.ssh/known_hosts. If the key changes later, SSH warns of potential MITM attack.

SSH Authentication Methods

üîë Autentificare cu Cheie PublicƒÉ

Client proves ownership of private key by signing a challenge. Most secure and recommended.

$ ssh-keygen -t ed25519

üîê Password Authentication

Traditional password through encrypted channel. Vulnerable to brute-force; often disabled.

üì± Keyboard-Interactive

Server sends arbitrary prompts. Enables 2FA, TOTP codes, challenge-response.

üé´ Certificate Authentication

SSH certificates signed by CA. Eliminates known_hosts management at scale.

Distribu»õia Cheilor

Copy your public key: ssh-copy-id user@server ‚Äî appends ~/.ssh/id_ed25519.pub to server's ~/.ssh/authorized_keys.

SSH Channel Multiplexing

A single SSH connection can carry multiple independent channels:

Practical Benefit

Multiplexing reduces connection overhead. With ControlMaster in SSH config, subsequent connections reuse the existing channel‚Äîmaking git pull or rsync faster.

Redirec»õionare Port: Local (-L)

Access remote services through an encrypted tunnel:

Use Case

Access a database server only reachable from within a private network, through a bastion host.

Syntax

-L [bind_address:]port:host:hostport

Traffic to local port forwards to host:hostport.

Redirec»õionare Port: Remote (-R)

Expose local services to the remote network:

Use Case

Share your local dev server with external testers or receive webhooks without deploying. Similar to ngrok.

Security NotƒÉ

Requires GatewayPorts yes in server's sshd_config to bind on public interfaces.

Redirec»õionare Port: Dynamic (-D) SOCKS Proxy

Creates a local SOCKS5 proxy that routes all traffic through the SSH server:

$ ssh -D 1080 user@proxy-server # Configure browser to use SOCKS5 # localhost:1080

Applications

  • Bypass geo-restrictions
  • Access internal networks
  • Privacy on untrusted networks

Common Pitfalls and Debugging

FTP: Mod Pasiv Port Blocked

Server PASV port range blocked by firewall. Configure both FTP server's passive range AND firewall rules.

DNS: Stale Cache

Changed records but seeing old IP? TTL hasn't expired. Use dig +trace to bypass cache.

SSH: Host Key Changed

Aten»õie could indicate server reinstall or MITM. Verify with admin before updating known_hosts.

SSH: Permission Denied

Check: ~/.ssh permissions (700), key file (600), authorized_keys (644), user in AllowUsers.
