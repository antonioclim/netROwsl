Enable WSL2

Requires Restart Windows

Windows Subsystem for Linux 2 (WSL2) provides a full Linux kernel running on Windows. This is the foundation that enables Docker containers to run natively without the overhead of Docker Desktop.

1 Open PowerShell as Administrator
  • Press Win + X or right-click the Start button
  • Select "Windows Terminal (Admin)" or "PowerShell (Admin)"
  • Click "Yes" on the User Account Control prompt
2 Install WSL2
PowerShell (Administrator) 
# Install WSL2 with default settings
wsl --install

This command enables WSL, enables the Virtual Machine Platform, downloads the Linux kernel and sets WSL2 as the default version.

3 Restart Your Computer
PowerShell 
Restart-Computer
‚ö†Ô∏è A restart is mandatory. Save all your work before proceeding. WSL2 features require system-level changes that only take effect after restart.
4 Verify Installation
PowerShell (after restart) 
# Check WSL status
wsl --status

# Expected output includes:
# Default Version: 2
# Kernel version: 5.15.x.x-microsoft-standard-WSL2
Verification Checklist
  • wsl --status shows "Default Version: 2"
  • No error messages about virtualisation
  • Kernel version displays a 5.15.x version number

Install Ubuntu 22.04 LTS

Required Windows

Ubuntu 22.04 LTS (Jammy Jellyfish) is our distribution of choice. LTS stands for Long Term Support, providing security updates until April 2027. It has excellent Docker compatibility and comprehensive documentation.

1 Install Ubuntu
PowerShell (Administrator)

wsl --install -d Ubuntu-22.04 --web-download
2 Create User Account

After download completes, Ubuntu will launch automatically. When prompted for a username and password:

Ubuntu Initial Setup

Retype new password: stud
‚ö†Ô∏è CRITICAL: Use exactly stud for both username and password. These credentials are used throughout all 14 weeks of laboratory exercises.
üí° Note: The password will not display as you type ‚Äî this is standard Linux security behaviour. Just type stud and press Enter.
3 Verify Installation

Verification Checklist

Install Docker Engine

Required Ubuntu WSL

Docker Engine is the container runtime that will host all laboratory containers. We install it directly inside Ubuntu WSL rather than using Docker Desktop for better performance and learning value.

üí° Open Ubuntu Terminal: Type wsl in PowerShell or search for "Ubuntu" in the Start menu. All commands in this step run inside Ubuntu.
1 Update System Packages
Ubuntu Terminal (WSL) 
# Update package lists and upgrade existing packages
sudo apt update && sudo apt upgrade -y

Enter password stud when prompted for sudo.

2 Install Prerequisites
Ubuntu Terminal (WSL) 
# Install packages required for Docker repository
sudo apt install -y \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg \
    lsb-release \
    software-properties-common
3 Add Docker Repository
Ubuntu Terminal (WSL) 
# Add Docker's official GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Add Docker repository to sources
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
4 Install Docker Engine
Ubuntu Terminal (WSL) 
# Update package list with Docker repository
sudo apt update

# Install Docker Engine and Compose plugin
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
5 Configure Docker for Non-Root Usage
Ubuntu Terminal (WSL) 
# Add your user to the docker group
sudo usermod -aG docker $USER

# Apply group changes immediately (or log out and back in)
newgrp docker
6 Start Docker and Verify
Ubuntu Terminal (WSL) 
# Start Docker service
sudo service docker start

# Verify Docker is working (should run without sudo)
docker --version
docker run hello-world

# Verify Docker Compose
docker compose version
Verification Checklist
  • docker --version shows Docker version 28.x or higher
  • docker run hello-world succeeds without sudo
  • docker compose version shows Compose version 2.x
  • docker ps runs without permission errors

Deploy Portainer CE (Global Service)

Port 9000 RESERVED Ubuntu WSL

Portainer Community Edition provides a web-based GUI for managing Docker containers. It runs as a global service on port 9000 throughout all 14 weeks. Weekly laboratory scripts will never interfere with Portainer.

‚ö†Ô∏è CRITICAL: Portainer uses port 9000. This port is permanently reserved and must never be used in any weekly docker-compose.yml file. Week 14's TCP Echo service uses port 9090 instead.
1 Create Portainer Volume and Deploy
Ubuntu Terminal (WSL) 
# Create persistent volume for Portainer data
docker volume create portainer_data

# Deploy Portainer CE container (GLOBAL - runs permanently)
docker run -d \
    -p 9000:9000 \
    --name portainer \
    --restart=always \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -v portainer_data:/data \
    portainer/portainer-ce:latest
2 Access Portainer and Create Admin Account
  • Open a web browser on Windows
  • Navigate to: http://localhost:9000
  • Create administrator account with these credentials:
    • Username: stud
    • Password: studstudstud
  • Click "Get Started" to connect to the local Docker environment
‚ö†Ô∏è Time Limit: You must create the admin account within 5 minutes of first access. If you miss this window, you'll need to redeploy Portainer (see Troubleshooting section).
Verification Checklist
  • docker ps shows portainer container running on port 9000
  • http://localhost:9000 loads Portainer dashboard
  • Can log in with stud / studstudstud
  • Dashboard shows "local" Docker environment with running containers

Install Wireshark

Required Windows

Wireshark is the world's foremost network protocol analyser. It is installed on Windows (not in WSL) to capture traffic from Docker containers via the vEthernet (WSL) virtual interface.

1 Download and Install
  • Download from wireshark.org/download.html
  • Select "Windows x64 Installer"
  • Run the installer with default options
2 Install Npcap (Essential)
  • When prompted during installation, install Npcap ‚Äî this is required for packet capture
  • In the Npcap installer, select these options:
    • ‚òëÔ∏è Install Npcap in WinPcap API-compatible Mode
    • ‚òëÔ∏è Support raw 802.11 traffic
3 Verify Wireshark Setup
  • Launch Wireshark from the Start menu
  • Look for "vEthernet (WSL)" in the interface list
  • This is the interface where you will capture all Docker container traffic
üí° Tip: If "vEthernet (WSL)" doesn't appear, ensure WSL is running. Open a PowerShell window and type wsl, then restart Wireshark.
Verification Checklist
  • Wireshark launches without errors
  • vEthernet (WSL) interface is visible in the interface list
  • Can start a capture on the vEthernet (WSL) interface

Configure Docker Auto-start

Optional Ubuntu WSL

Configure Docker to start automatically when you open Ubuntu. This saves time and ensures Portainer is always accessible.

1 Edit .bashrc
Ubuntu Terminal (WSL) 
# Open .bashrc for editing
nano ~/.bashrc

# Add these lines at the end of the file:
# Auto-start Docker service
if service docker status 2>&1 | grep -q "is not running"; then
    sudo service docker start
fi

Save with Ctrl+O, Enter, then exit with Ctrl+X.

2 Configure Passwordless Docker Start
Ubuntu Terminal (WSL) 
# Allow Docker service start without password
echo "stud ALL=(ALL) NOPASSWD: /usr/sbin/service docker start" | sudo tee /etc/sudoers.d/docker-start

# Set correct permissions
sudo chmod 440 /etc/sudoers.d/docker-start
3 Test Auto-start
PowerShell 
# Shutdown WSL completely
wsl --shutdown

# Reopen Ubuntu
wsl

# Docker should start automatically ‚Äî verify with:
docker ps
Verification Checklist
  • Docker starts automatically when opening Ubuntu
  • No password prompt for Docker service
  • docker ps works immediately after opening terminal
  • Portainer accessible at http://localhost:9000

Common Issues and Solutions

"Cannot connect to the Docker daemon"

Docker service is not running. Start it manually:

Ubuntu Terminal 
sudo service docker start

"Permission denied" when running docker commands

User is not in the docker group. Add the user and apply changes:

Ubuntu Terminal 
sudo usermod -aG docker $USER
newgrp docker

Portainer: "Timeout" or "5-minute window expired"

Redeploy Portainer with a fresh volume:

Ubuntu Terminal 
docker stop portainer && docker rm portainer
docker volume rm portainer_data
docker volume create portainer_data
docker run -d -p 9000:9000 --name portainer --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data portainer/portainer-ce:latest

Then access http://localhost:9000 immediately and create the admin account.

Wireshark: "vEthernet (WSL)" not visible

WSL must be running for the interface to appear. Open Ubuntu terminal first, then restart Wireshark.

WSL won't start or is slow

Reset WSL and ensure virtualisation is enabled in BIOS:

PowerShell 
wsl --shutdown
wsl
