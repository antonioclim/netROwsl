This guide uses WSL2 + Docker inside Ubuntu (not Docker Desktop). Some steps require restarts. Complete each step before proceeding to the next.

Enable WSL2

Requires Restart Windows

Windows Subsystem for Linux 2 provides a full Linux kernel running on Windows. This is the foundation for running Docker containers natively.

  • Open PowerShell as Administrator (right-click → Run as administrator)
  • Run the installation command shown below
  • Restart your computer when prompted
PowerShell (Administrator) 
# Install WSL2 (requires restart)
wsl --install

# After restart, verify installation
wsl --status
Verification Checklist
  • wsl --status shows "Default Version: 2"
  • WSL is listed as enabled
2

Install Ubuntu 22.04

Required Windows

Ubuntu 22.04 LTS (Jammy Jellyfish) provides a stable, long-term supported Linux distribution for running Docker and laboratory tools.

  • Open PowerShell as Administrator
  • Install Ubuntu 22.04 using the command below
  • When prompted, create user: stud with password: stud
PowerShell (Administrator) 
# Install Ubuntu 22.04
wsl --install -d Ubuntu-22.04 --web-download

# List installed distributions
wsl -l -v
⚠️ Important: When creating the UNIX user, use username stud and password stud for consistency across all laboratory exercises.
Verification Checklist
  • wsl -l -v shows Ubuntu-22.04 with VERSION 2
  • Ubuntu launches with user stud
3

Install Docker in WSL

Required Ubuntu WSL

Docker provides containerisation for running isolated network environments. We install Docker directly in Ubuntu WSL (not Docker Desktop) for better performance and native Linux integration.

  • Open Ubuntu from the Start menu or type wsl in PowerShell
  • Run the commands below to update the system and install Docker
  • Add your user to the docker group to run commands without sudo
Ubuntu Terminal (WSL) 
# Update system packages
sudo apt update && sudo apt upgrade -y

# Install Docker and Docker Compose
sudo apt install -y docker.io docker-compose

# Add user to docker group (run without sudo)
sudo usermod -aG docker $USER

# Start Docker service
sudo service docker start

# Apply group changes (or log out and back in)
newgrp docker
Verify Installation 
# Check Docker version
docker --version

# Test Docker (should work without sudo now)
docker run hello-world
Verification Checklist
  • docker --version shows Docker version 28.x or higher
  • docker run hello-world succeeds without sudo
  • docker ps runs without permission errors
4

Deploy Portainer CE

Required Ubuntu WSL

Portainer Community Edition provides a web-based graphical interface for managing Docker containers, images, volumes, and networks. Access it from your Windows browser.

  • Run the commands below in your Ubuntu WSL terminal
  • Access Portainer at http://localhost:9000 from Windows
  • Create admin account: username stud, password studstudstud
  • Select "Get Started" to connect to the local Docker environment
Ubuntu Terminal (WSL) 
# Create persistent volume for Portainer data
docker volume create portainer_data

# Deploy Portainer CE container
docker run -d -p 9000:9000 --name portainer --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer-ce:latest

# Verify Portainer is running
docker ps
⚠️ First Login: When you first access http://localhost:9000, you must create an admin account within 5 minutes. Use:
Username: stud  |  Password: studstudstud (minimum 12 characters)
Verification Checklist
  • docker ps shows portainer container running
  • http://localhost:9000 loads in Windows browser
  • Can log in with stud/studstudstud credentials
  • Dashboard shows "local" Docker environment connected

Install Wireshark

Required Windows

Wireshark is the world's most popular network protocol analyser. Install it on Windows to capture and analyse traffic from Docker containers running in WSL.

  • Download from wireshark.org/download.html
  • Select "Windows x64 Installer"
  • Run the installer with default options
  • Important: When prompted, install Npcap (required for packet capture)
  • During Npcap installation, check "Support raw 802.11 traffic" and "Install Npcap in WinPcap API-compatible Mode"
Verification Checklist
  • Wireshark launches from Start menu
  • Network interfaces are visible (including "vEthernet (WSL)")
  • Can start a capture on "vEthernet (WSL)" interface
  • Traffic from Docker containers is visible when running

Python Packages (Windows)

Recommended Windows

Install Python packages on Windows for running lab scripts that interact with Docker. Ensure Python 3.11+ is installed on your system.

PowerShell / Command Prompt 
# Required package for Docker control
pip install docker

# Network analysis packages (optional but recommended)
pip install scapy dpkt

# Verify installation
python -c "import docker; print('Docker SDK: OK')"
Verification
  • pip show docker displays package information
  • python -c "import docker; print('OK')" prints OK

Configure Auto-start (Optional)

Recommended Ubuntu WSL

Configure Docker to start automatically when you open Ubuntu WSL. This saves time by eliminating the need to manually start the Docker service.

Ubuntu Terminal (WSL) 
# Add Docker auto-start to .bashrc
echo '# Auto-start Docker service' >> ~/.bashrc
echo 'if ! pgrep -x "dockerd" > /dev/null; then' >> ~/.bashrc
echo '    sudo service docker start > /dev/null 2>&1' >> ~/.bashrc
echo 'fi' >> ~/.bashrc

# Allow docker service start without password
sudo visudo
📝 In visudo, add this line at the end:
stud ALL=(ALL) NOPASSWD: /usr/sbin/service docker start

Press Ctrl+X, then Y, then Enter to save and exit.

🔍 Quick Verification Commands

Run these commands to verify your complete setup:

Ubuntu Terminal (WSL)

echo "=== WSL Status ===" && wsl.exe --status

✅ Final Verification Test

Test that Wireshark can capture Docker container traffic. Start a capture on vEthernet (WSL) interface, then run a ping from a container:

Ubuntu Terminal (WSL) 
# Generate test ICMP traffic from a Docker container
docker run --rm alpine ping -c 5 8.8.8.8

In Wireshark, filter with icmp to see the ping packets:

Example: ICMP Echo Request/Reply packets between Docker container (172.27.159.165) and Google DNS (8.8.8.8)
