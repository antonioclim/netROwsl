# Code of Conduct

## Purpose

This Code of Conduct establishes expectations for participation in the netENwsl project community, ensuring a respectful and productive environment for all students, educators and contributors.

## Our Standards

### Expected Behaviour

Participants in this project are expected to:

| Behaviour | Description |
|-----------|-------------|
| **Be Respectful** | Treat all participants with courtesy, regardless of experience level |
| **Be Constructive** | Provide helpful feedback; focus on problems, not people |
| **Be Patient** | Remember that learners have varying backgrounds and abilities |
| **Be Collaborative** | Share knowledge willingly; help others when you can |
| **Be Professional** | Maintain appropriate language and conduct at all times |
| **Be Honest** | Acknowledge your sources; do not present others' work as your own |

### Academic Integrity

As an educational project, we emphasise:

- **Original Work**: Complete exercises yourself before consulting solutions
- **Proper Attribution**: Credit sources when incorporating external ideas
- **Honest Reporting**: Report bugs and issues accurately
- **Respect for Licence**: Honour the restrictive educational licence terms

### Unacceptable Behaviour

The following behaviours are not tolerated:

| Behaviour | Examples |
|-----------|----------|
| **Harassment** | Offensive comments, personal attacks, unwelcome attention |
| **Discrimination** | Exclusion based on personal characteristics |
| **Disruption** | Trolling, spam, off-topic disruption of discussions |
| **Academic Dishonesty** | Plagiarism, sharing solutions inappropriately |
| **Licence Violations** | Redistributing materials contrary to licence terms |
| **Inappropriate Content** | Offensive, violent or otherwise unsuitable material |

## Scope

This Code of Conduct applies to:

- GitHub Issues and Pull Requests
- Repository discussions
- Any other project-related communications
- Representations of the project in public spaces

## Enforcement

### Reporting

If you witness or experience unacceptable behaviour:

1. **Minor Issues**: Comment directly, assuming good faith (misunderstanding may be involved)
2. **Serious Issues**: Report to the repository maintainers through GitHub

Include in your report:
- What happened
- When and where it occurred
- Any relevant context

### Consequences

Maintainers may take the following actions:

| Violation Level | Response |
|-----------------|----------|
| First minor offence | Private warning with explanation |
| Repeated minor offences | Public warning on the issue/PR |
| Serious offence | Temporary ban from participation |
| Severe or repeated serious offences | Permanent ban |

### Appeals

If you believe enforcement action was taken in error, you may appeal by contacting the maintainers with a clear explanation of your perspective.

## Maintainer Responsibilities

Project maintainers commit to:

- Applying this Code of Conduct fairly and consistently
- Responding to reports promptly
- Maintaining confidentiality of reporters
- Explaining decisions when appropriate
- Leading by example in their own conduct
